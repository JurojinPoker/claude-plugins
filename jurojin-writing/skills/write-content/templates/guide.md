# Template: How-to / help guide (format-neutral)

Describes the *structure and voice* of a guide. Rendering comes from the repo's
writing-profile.

## Shape

- **Title** — task-oriented, starts with a verb ("Install Jurojin", "Redeem a
  gift card"). Match how a user would search for it.
- **One-line intro** — what they'll achieve and, if relevant, prerequisites.
- **Steps** — numbered, one action per step, imperative mood. Each step should
  be verifiable ("you'll see X").
- **Callouts** — warnings/tips only where a user genuinely gets stuck. Don't
  over-decorate.
- **Result / verification** — how the user confirms it worked.
- **Related / next** — the logical next guide or a relevant FAQ.

## Voice

- Calm, precise, second person. Assume a non-technical player.
- Use canonical product terms from the brand glossary consistently.
- Prefer screenshots/visual anchors where the repo profile supports them
  (describe the image intent with a `TODO:` if the asset doesn't exist yet).

## Quality bar

- A new user should complete the task without external help.
- No step should require knowledge introduced only in a later step.
