# Template: Release / news post (format-neutral)

Describes the *structure and voice* of a release or news post. Rendering comes
from the repo's writing-profile.

## Shape

- **Headline** — what's new, in user-benefit terms, not version numbers.
  ("Faster hand history sync" beats "v2.4.0 released").
- **Hook (1–2 sentences)** — why this matters to the player, up front.
- **What changed** — the concrete changes, as a short list. Group as
  *New / Improved / Fixed* when there are several.
- **Why it matters** — the payoff in the user's day-to-day, briefly.
- **CTA** — one clear next action (update, try it, read the guide). See the
  brand doc's CTA library for canonical wording.

## Voice

- Energetic but honest; no overpromising. Lead with benefit, name the feature
  second.
- Credit fixes plainly; don't bury or spin known issues that were resolved.

## Length

- Keep it skimmable. Most release posts are a hook + a bulleted list + a CTA.
  Save long-form for guides.
